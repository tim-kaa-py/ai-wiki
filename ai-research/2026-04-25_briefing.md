## Claude Code Updates

*Source: [@claudelog](https://www.youtube.com/@claudelog) — 5 most recent videos*

### [v2.1.120 — ultrareview in CI](https://www.youtube.com/watch?v=_8cOVcgTIeU)

- **`/ultrareview` runs headless in CI.** You can now trigger a deep multi-agent cloud code review without an interactive session. Point it at a branch or PR, pipe findings with `--output json`, and gate your pipeline on the exit code (0 = complete, 1 = failed review).
  - **So what:** Wire `/ultrareview` into your CI pipeline to block merges on significant issues — no engineer needs to be online for it to run.
- **Windows: Git Bash no longer required.** Claude Code now falls back to PowerShell automatically when Git Bash is absent.
  - **So what:** Windows users can run Claude Code natively without any shell setup step.
- **Bug fix: file descriptor exhaustion on large repos.** A bug where the `find` tool could exhaust all open file descriptors — taking down the host process — is patched.
  - **So what:** Large monorepos are safe again; this was a hard crash on affected systems.

---

### [v2.1.119 — Settings That Actually Stick](https://www.youtube.com/watch?v=rkac5fIfVz4)

- **Config settings now persist.** Theme, editor mode, verbose flag — set once via `/config`, saved to `~/.claude/settings.json`, survive restarts. The full project → local → policy override hierarchy is respected.
  - **So what:** You no longer need to re-apply your preferences every session.
- **`-r` / `--review` now works with GitLab, Bitbucket, and GitHub Enterprise.** Pass any merge request or PR URL and Claude picks it up regardless of platform.
  - **So what:** Teams not on github.com can now use the PR review workflow without workarounds.

---

### [v2.1.118 — Vim Visual Mode & Custom Themes](https://www.youtube.com/watch?v=hPeNrtzvCig)

- **Vim visual mode is here.** Press `v` for character-wise visual mode, `V` for line mode. Operators, selections, and visual feedback all work as expected.
  - **So what:** If you use Vim keybindings, text selection in the terminal editor now works the way your muscle memory expects.
- **Full custom theming.** Create and switch named themes via `/theme`. Hand-edit JSON in `~/.claude/themes/` for full control. Plugins can ship their own themes via a `themes/` directory.
  - **So what:** You can fully style the Claude Code UI — either personally or distribute a team theme via a plugin.

---

### [v2.1.117 — Opus 4.7 Context Window Fixed, Faster Search](https://www.youtube.com/watch?v=X13MxJNLRag)

- **Opus 4.7 context window corrected.** Sessions were auto-compacting far too early because Claude Code was calculating context against a 200k window — but Opus 4.7's real limit is 1M tokens. Now fixed.
  - **So what:** If you use Opus 4.7, your sessions will run significantly longer before hitting compaction. Context percentage displays and compact triggers now reflect reality.
- **Glob and grep replaced by embedded BFS + ugrep (macOS/Linux native builds).** These now run directly through the Bash tool with no separate tool round trip.
  - **So what:** File searches are faster. Windows and npm installs are unchanged.

---

### [v2.1.111 — Opus 4.7 xhigh Effort & /ultrareview Launch](https://www.youtube.com/watch?v=pcRkn0QTED4)

- **New effort tier: `xhigh`.** Sits between `high` and `max`. Run `/effort` with no arguments for an interactive slider — arrow keys to select, Enter to confirm.
  - **So what:** Use `xhigh` for hard tasks where `high` undershoots but `max` is overkill. Finer-grained cost/quality control.
- **`/ultrareview` introduced.** Runs parallel cloud agents to review your current branch or a specific PR number. This is a senior engineer-style review, not a linter.
  - **So what:** Get a comprehensive code review without another human — useful before important merges or when working solo.
